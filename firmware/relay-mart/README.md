# relay-mart — боевая прошивка «оплата → открытие реле»

ESP-IDF прошивка для платы реле (таргет **esp32**, плата ESP32-D0WD).
Работает как `esp-relay`, но с **защёлкивающимся реле** (IO2/IO16) и открытием на **20 секунд**.

## Что делает
1. WiFi + провижининг через SoftAP-портал (как esp-relay): вводите WiFi и Machine ID,
   прошивка тянет MQTT-идентичность по `device-provision` и сохраняет в NVS.
2. Подключается к `mqtt.smartvend.kz:14003`, подписывается на `vending/<uuid>/in`.
3. На сообщение оплаты `{"msg":"Processed order","code":1,"orderid":...}`:
   - защёлкивает реле (ВКЛ),
   - держит **20 секунд** (`OPEN_SECONDS`),
   - размыкает (ВЫКЛ),
   - шлёт `complete-order` в Supabase (с ретраями).
4. Защита от повторов: один `orderid` не открывает дважды (хранится в NVS).

## Время открытия
Меняется одной константой в [main/main.c](main/main.c):
```c
#define OPEN_SECONDS   20
```
Реле бистабильное и держит положение без тока; время — программный таймер
(FreeRTOS-задача `trigger_lock`), независимый от сети.

## Пины реле (из реверса штатной прошивки)
- DIR = **IO2**, PULSE = **IO16** (канал 0). Проверены на стенде через `relay-test`.

## Сборка и прошивка
> ⚠️ Перезапишет текущую прошивку. Бэкап штатной: `../../esp32_dump/flash_full.bin`.
```bash
cd firmware/relay-mart
idf.py set-target esp32
idf.py -p COM9 build flash monitor
```

## Первая настройка (провижининг)
1. Включите/перезагрузите плату и в **первые 4 секунды нажмите кнопку BOOT (GPIO0)**.
   Поднимется точка `SmartVend-Setup-XXXX`. (На чистом устройстве без конфига
   портал стартует сам.)
2. Подключитесь к ней телефоном — откроется страница настройки.
3. Введите WiFi и **Machine ID** (номер аппарата в вашей системе).
4. Устройство проверит WiFi, получит MQTT-идентичность и перезагрузится в рабочий режим.

После этого любая подтверждённая оплата открывает реле на 20 секунд.
