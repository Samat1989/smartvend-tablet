# Интеграция вендинг-платы LiYuTai (V27.2 / STC8) — финальная шпаргалка

Полный контракт управления платой, **проверенный на живом железе** (выдача по микрику,
статус 0 = успех). Источник: реверс заводского APK `com.li.fut` («Perfume»,
Unity/Mono, класс `ThreadManager`) + отладка на стенде и в автомате.

---

## 1. Архитектура

Планшет ↔ плата: последовательный порт **`/dev/ttyS1`, 115200 8N1**.
Плата — общий спиральный вендинг-борт: матрица моторов `L×C` + обратная связь микриков `K`.

```
[Планшет/контроллер] --UART 115200--> [Плата V27.2 STC8] --L×C--> моторы спиралей
                                                          <--K---- микрики (home)
                                                          <------- монетоприёмник
```

---

## 2. Аппаратное подключение (обязательно!)

| Узел | Куда | Примечание |
|------|------|-----------|
| Питание логики | пин **`12V`** + `GND` | через `FUSE=3A` → buck 5В/3.3В |
| Питание моторов | пин **`12/24V`** + `GND` | **24 В** для нормальной скорости |
| **Привод мотора** | 2 провода → матрица **`L1..L10`** (ряд) × **`C1..C14`** (колонка) | плата подаёт на L[ряд] и C[кол] |
| **Микрик / home** | 3-й провод мотора → разъём **`K1..K13`** (货道) | **БЕЗ него плата не видит «дом» → перекрут** |
| Связь | пин `TXD`/`RXD` разъёма **`TTL`** (или через планшетный UART) | 115200 8N1 |

> ⚠️ **Мотор 3-проводной:** 2 привода (L×C) + 1 микрик (K). Микрик по ряду:
> ряд 1 → `K1`, ряд 2 → `K2`, … Точное соответствие пинов сверьте по жгуту автомата.

---

## 3. Формат кадра

```
AA [len] [ ... ] [XOR] DD
```
- старт `0xAA`, стоп `0xDD`;
- **контрольная сумма = XOR всех байтов от индекса 1 до байта перед XOR** (включая байт длины).

```c
uint8_t xor_sum(const uint8_t *f, int from, int to) { // to включительно
    uint8_t x = 0;
    for (int i = from; i <= to; i++) x ^= f[i];
    return x;
}
```

---

## 4. Команда ВЫДАЧИ (планшет → плата) — 17 байт

```
AA 0E 01 30 [ряд] [кол] [t_hi] [t_lo] 00 00 00 00 01 00 00 [XOR] DD
```

| Индекс | Значение | Смысл |
|-------:|----------|-------|
| 0 | `AA` | старт |
| 1 | `0E` | длина |
| 2 | `01` | команда = выдача |
| 3 | `30` | константа |
| 4 | `ряд` | ряд (L), 1-based |
| 5 | `кол` | колонка (C), 1-based |
| 6 | `t_hi` | таймаут, старший байт (16 бит **big-endian**) |
| 7 | `t_lo` | таймаут, младший байт |
| 8..11 | `00` | резерв |
| 12 | `01` | константа (perfume_time) |
| 13,14 | `00` | резерв |
| 15 | `XOR` | XOR байтов [1..14] |
| 16 | `DD` | стоп |

> **«Время» — это ТАЙМАУТ-защита, а не длительность.** Плата крутит мотор до срабатывания
> микрика (дом), а таймаут — предел ожидания. Держите с запасом: **600–800**.
> Если микрик не пришёл за таймаут → ошибка (статус 2).

**Пример — ряд 1, кол 1, таймаут 200 (0x00C8):**
```
AA 0E 01 30 01 01 00 C8 00 00 00 00 01 00 00 F6 DD
```
XOR[1..14] = `0E^01^30^01^01^00^C8^00^00^00^00^01^00^00 = 0xF6` ✔

**Псевдокод сборки:**
```
frame = [0xAA,0x0E,0x01,0x30, row, col, (t>>8)&0xFF, t&0xFF, 0,0,0,0, 0x01, 0,0]
frame += [ xor_sum(frame,1,14), 0xDD ]
uart_write(frame)   // /dev/ttyS1 @115200
```

---

## 5. Ответ — РЕЗУЛЬТАТ ВЫДАЧИ (плата → планшет) — 17 байт

```
AA 0E 02 30 [ряд] [кол] [t_hi] [t_lo] 00 00 00 00 [СТАТУС] 00 00 [XOR] DD
```
- отличие от запроса: **`[2]=02`**;
- **`[4][5]` — эхо ряд/кол** (по нему знаете, какая выдача завершилась);
- **`[12]` = СТАТУС:**

| [12] | Значение |
|------|----------|
| **`00`** | **успех — товар выдан** ✅ |
| `01` | ошибка — мотор не сработал |
| `02` | ошибка — товар не зафиксирован (не дошёл до дома / таймаут) |

**Реальный ответ-успех (стенд, ряд 6 кол 6):**
```
AA 0E 02 30 06 06 00 C8 00 00 00 00 00 00 00 F4 DD   → [12]=00 = успех
```

> ⚠️ В ЗАПРОСе `[12]=01` (perfume_time), в ОТВЕТе `[12]=статус`. Не перепутайте.

---

## 6. Приём монет (плата шлёт сама при оплате) — 7 байт

```
AA 03 30 [cnt_hi] [cnt_lo] [XOR] DD
```
- `[1]=03`, `[2]=30`; XOR по байтам [1..4];
- **накопительный счётчик = `cnt_hi*256 + cnt_lo`**;
- берите **дельту** относительно прошлого значения; **сумма = дельта × номинал_монеты**.

---

## 7. Прочие команды (справочно)

| Команда | Направление | Кадр | Примечание |
|---------|-------------|------|-----------|
| Версия платы | план→плата | `AA 02 80 82 DD` | **эта плата (V27.2) НЕ отвечает** — не использовать для проверки связи |
| OTA прошивки | план→плата | пакеты, cmd `0xCD` | только при обновлении прошивки платы |

Для «жив ли порт/плата» используйте **саму выдачу** (плата на неё отвечает), а не версию.

---

## 8. Разбор входящего потока (де-фрейминг)

Плата отвечает **только по событию** (выдача/монета), периодически молчит.
Собирайте кадры из потока по правилу «AA … XOR DD»:

```
буфер += принятые байты
цикл:
  отбросить байты в начале, пока буфер[0] != 0xAA
  если известный кадр результата (>=17б, [1]=0E,[2]=02,[3]=30,[16]=DD, XOR[1..14]==[15]) → разобрать, срезать 17
  иначе если кадр монет (>=7б, [1]=03,[2]=30,[6]=DD, XOR[1..4]==[5]) → разобрать, срезать 7
  иначе искать кратчайший валидный AA..DD с верным XOR
  если валидного нет и байтов мало → ждать ещё
```
Готовая реализация: `LytBoardClient._drain()` в
[lyt_client.dart](mmd_diag/lib/lyt_client.dart).

## 9. Поиск порта платы (автоскан)

Если не известно, на каком `ttyS` и какой скорости висит плата — автоскан, как в
диагностическом приложении `mmd_diag` (реализация: `NativeUartTransport.listPorts()` в
[transport.dart](mmd_diag/lib/transport.dart) + `LytBoardClient.scanPorts()` в
[lyt_client.dart](mmd_diag/lib/lyt_client.dart)).

### 9.1. Перечисление узлов портов

Android не даёт Dart/Java termios-API для нативных UART, поэтому всё через coreutils
устройства. Сначала находим **работающий провайдер команд** (первый, у кого `ls /dev`
вернул 0):

```
/system/xbin/busybox → /system/bin/busybox → /vendor/bin/busybox
→ /system/bin/toybox → /vendor/bin/toybox → прямые команды из PATH
```

> ⚠️ Не хардкодить `/system/xbin/busybox` — на части планшетов его нет, и поиск
> молча ничего не находил, пока не добавили перебор провайдеров.

Затем узлы собираются **двумя способами** (результаты объединяются):

1. **Прямой листинг `/dev`** с фильтром по regex `^tty(S|USB|ACM)\d+$`
   (на Android листинг может быть запрещён SELinux — исключение ловим и идём дальше);
2. **Проба фиксированных кандидатов** через провайдер: `ls /dev/ttyS0..9`,
   `/dev/ttyUSB0..7`, `/dev/ttyACM0..3` — узел существует, если `ls` вернул 0.

USB-адаптеры (CH340/FTDI) — отдельная ветка: перечисляются плагином `usb_serial`
(`UsbSerial.listDevices()`), скан ниже к ним не применяется.

### 9.2. Активная проба (где именно плата)

Для каждого найденного узла × каждой скорости (**115200, затем 9600**):

```
открыть порт (stty raw 8N1 → cat/dd)
  не открылся            → «занят» (порт держит заводское приложение)
отправить ПРОБУ = кадр ВЫДАЧИ, ряд 1 кол 1, таймаут ≈10
слушать ~1.5 с, копить байты
закрыть, классифицировать:
  0 байт                     → тишина (не та линия / плата не отреагировала)
  валидный кадр AA..DD (XOR) → ПЛАТА НАЙДЕНА ✓  (порт + скорость известны)
  все байты печатные ASCII   → чья-то debug-консоль, не плата
  прочее                     → мусор — данные есть, но скорость не та
```

Валидность кадра: в буфере ищется `0xAA`, для каждого `0xDD` в пределах 64 байт
проверяется `XOR[start+1 .. end-2] == байт перед DD` (см. `_containsFrame()`).

> ⚠️ **Проба — настоящая команда выдачи**: на живой плате мотор ряд 1/кол 1
> крутанётся. Это осознанно — версия (`0x80`) и heartbeat платой V27.2
> игнорируются (§7), выдача — единственная команда, по ответу на которую плату
> видно достоверно. Таймаут в пробе ставим маленький (~10), чтобы мотор только
> дёрнулся.
>
> ⚠️ Скан нельзя запускать при активном подключении, и он честно покажет «занят»,
> если порт держит заводское ПО (§11 п.5) — сначала остановить его.

## 10. Рецепт интеграции (в боевое приложение)

```
// однократно
open("/dev/ttyS1", 115200, 8N1)
start_rx_thread()        // слушает монеты и результаты выдачи

// выдать товар из (ряд, кол)
send_dispense(row, col, timeout=700)
result = wait_response(AA 0E 02 30 ..., timeout=timeout+буфер)
if result[12] == 0:  списать остаток, показать «выдано»
else:                показать ошибку / повторить

// монеты — асинхронно из rx_thread
on_coin(delta): баланс += delta * номинал_монеты
```

Адресация — **пара (ряд, колонка)**. Если у вас мотор по одному id — сделайте маппинг
`id → (ряд, кол)` под физическую раскладку автомата.

## 11. Ключевые уроки (грабли, которые прошли)

1. **Мотор 3-проводной** — без микрика в `K` идёт перекрут (крутит до таймаута). Обязательно завести home-провод.
2. **«Время» = таймаут**, а не длительность. Стоп — по микрику. Держите таймаут с запасом.
3. **24 В на моторы** (12 В — медленно, не докручивает до дома → статус 2).
4. **Уровень UART платы низкий (~1.8/3.3 В)** — внешний 3.3/5 В FTDI читает нестабильно. Управлять с родного порта планшета (совпадает) или через преобразователь/логический анализатор.
5. **Порт эксклюзивный** — заводское приложение держит `/dev/ttyS1`; своё ПО не откроет порт, пока заводское работает. Останавливать заводское.
6. **Версия платы (`0x80`) не отвечает** — для проверки связи слать выдачу.
7. Управление моторами — **отдельный протокол на `ttyS1`**, платёжный Marshall/MDB на `ttyS5` в этой сборке **мёртв** (не инициализируется). Облачный слой — **MQTT** (удалённая выдача/оплата), задаётся в рантайме.
