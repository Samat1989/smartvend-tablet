# 📜 Development Rules & Standards (Kiosk App)

### 🔐 Authentication & Security
- **Secure Persistence:** Логин и пароль аппарата должны сохраняться безопасно (используйте Android `EncryptedSharedPreferences` / Android Keystore).
- **Auto-Login:** Никогда не заставляйте покупателя видеть процесс загрузки или логин. Восстановление сессии Supabase должно быть фоновым на этапе Splash-скрина.

### 🚀 Android Kiosk Mode 
- **Lock Task Mode:** Обязательно вызывать `startLockTask()`, если устройство находится в режиме Device Owner. Это предотвратит выход из приложения.
- **Immersive Mode:** Строго скрывать Navigation Bar и Status Bar (`WindowCompat.setDecorFitsSystemWindows`).
- **Сброс при бездействии:** Обязательная реализация UI-таймаута. Если экран не трогают N секунд (например, 30 или 60) — корзина сбрасывается и UI возвращается в дефолтное состояние (Screen Saver).

### 🔢 Synchronization (Database)
- **Целостность данных:** Оплата по QR $\rightarrow$ создание записи в `sales` $\rightarrow$ уменьшение `stock` в `inventory`. Эта цепочка должна быть атомарной и надежной. Желательно использовать Supabase RPC для проведения транзакции, либо транзакционно писать в 2 таблицы.
- **Оффлайн отказоустойчивость:** Если планшет теряет Wi-Fi, QR оплата может не работать. Если оплата прошла, а интернет пропал перед записью в Supabase, чек должен сохраниться в SQLDelight/Room локально и отправиться позже.

### 📦 UI / UX 
- **Адаптация под планшет:** Элементы интерфейса (плитки товаров, кнопки плюса/минуса, кнопка оплаты) должны быть очень крупными и предназначаться для работы грязными/в перчатках руками.
- **Защита от двойных нажатий:** Блокировать кнопку корзины (Debounce) при оплате, чтобы не порождать двойные QR.
