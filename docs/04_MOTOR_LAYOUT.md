# Раскладка моторов — конкретный аппарат

> Получено эмпирически: запуск `python vending_ctrl.py --port COM6 --motor N --type 2` по каждому индексу и наблюдение какая ячейка крутится.

## Тип моторов

**Двухконтактные (2-проводные)** — `--type 2`.

Без обратной связи; плата останавливает мотор по таймауту (~6-8 сек). С `type=3` (трёхпроводной режим) такие моторы вернут ошибку 10 (микровыключатель не нажат), либо поведение будет некорректным.

## Два числовых пространства

Важно различать:

- **Подпись на полке** (что видит оператор: `001`, `011`, `021`, ..., `056`) — нумерация от производителя, для физической ориентации.
- **Индекс протокола** (что отправляется в команде `0x05`: `44..99`) — внутренний индекс мотора на плате M102.

Они **разные** и не совпадают. В документации и в новом приложении используйте подписи на полке (их видит оператор), а в драйвер-слое транслируйте в индексы протокола.

## Формулы

### Координаты ↔ индекс протокола

```
motor_id = (10 - row) * 10 + (10 - col)
row = 10 - (motor_id // 10)
col = 10 - (motor_id % 10)
```
где `row` = 1..6 (сверху вниз), `col` = 1..6 (слева направо).

### Подпись на полке ↔ координаты

Подпись имеет вид `0RC` где `R` — номер уровня (0..5, сверху вниз), `C` — номер ячейки в строке (1..6, слева направо):

```
shelf_label = (row - 1) * 10 + col      # row,col → label
row = (shelf_label // 10) + 1           # label → row
col = shelf_label % 10                  # label → col
```

### Подпись ↔ индекс протокола (напрямую)

```
motor_id = (9 - shelf_label // 10) * 10 + (10 - shelf_label % 10)
```

Проверка:
- Подпись `001` (row=1, col=1) → `(9-0)*10 + (10-1)` = **99** ✓
- Подпись `006` (row=1, col=6) → `(9-0)*10 + (10-6)` = **94** ✓
- Подпись `011` (row=2, col=1) → `(9-1)*10 + (10-1)` = **89** ✓
- Подпись `056` (row=6, col=6) → `(9-5)*10 + (10-6)` = **44** ✓

## Логика конструктора

Заводская плата M102 поддерживает индексы **0-99** для главной секции (адрес `01`). Производитель аппарата:
- Использует только **верхние 6 десятков** (4_, 5_, 6_, 7_, 8_, 9_)
- В каждом десятке использует **последние 6 индексов** (X4-X9)
- Десятки 0-3 и индексы X0-X3 в пределах десятка **не задействованы**

Скорее всего это сделано чтобы:
- Зарезервировать индексы X0-X3 под замки/служебные выходы (60-99 в протоколе = "замки")
- Оставить место под расширение секций без перенумерации

## Команды для тестирования

```bash
# Самый верхний левый (99)
python vending_ctrl.py --port COM6 --motor 99 --type 2

# Самый нижний правый (44)
python vending_ctrl.py --port COM6 --motor 44 --type 2

# Прогнать всю верхнюю строку
for n in 99 98 97 96 95 94; do
    python vending_ctrl.py --port COM6 --motor $n --type 2
    sleep 8  # ждать завершения мотора + запас
done
```

## Маппинг для нового приложения

В новом приложении удобнее работать в координатах `(row, col)` или давать ячейкам человеческие имена `A1..F6`:

```python
# Для нового приложения
def coords_to_motor(row, col):
    """row: 1..6 (1=top), col: 1..6 (1=left) → motor_id 44..99"""
    return (10 - row) * 10 + (10 - col)

def motor_to_coords(motor_id):
    """motor_id 44..99 → (row, col)"""
    return 10 - motor_id // 10, 10 - motor_id % 10

def label_to_motor(label):
    """shelf label '001'..'056' → motor_id 44..99"""
    n = int(label)
    return (9 - n // 10) * 10 + (10 - n % 10)

def motor_to_label(motor_id):
    """motor_id 44..99 → shelf label string '001'..'056'"""
    row, col = motor_to_coords(motor_id)
    return f"{(row - 1) * 10 + col:03d}"
```

При сохранении в БД (Room/SQLite) рекомендую такие поля:
- `shelf_label` — `"001"`, `"023"` и т.д. (то что подписано на полке — оператор будет искать ячейку именно по этой надписи)
- `row`, `col` — int, int (для сетки UI)
- `motor_id` — int (для команды плате, 44..99)
- `motor_type` — int (2 у вас, поддержать и 3 на будущее)

## Замечания

- **Привязка ячейка → товар хранится в `GoodEntity.HD`** в заводской БД как 4 hex-цифры (cabinet + motor). У вас всё в одной секции `01`, значит `HD = "01" + HEX(motor_id)` (например, мотор 99 → `HD = "0163"`).
- При замене приложения **не используйте 4-hex формат** заводской БД — он избыточен. Делайте свою схему.
- Если позже добавится **вторая секция** (副柜) — в формуле появится поле `cabinet` (`01` главная, `02` боковая 1 и т.д.).
