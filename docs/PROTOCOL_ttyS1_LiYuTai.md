# Протокол управления вендинг-платой (LiYuTai / STC8C8K64U)

**Источник:** реверс заводского APK `com.li.fut` («Perfume 1.1.1.1.3», производитель LiYuTai).
Приложение — **Unity (Mono)**; логика управления в `Assembly-CSharp.dll` → класс `ThreadManager`.
Плата управления автоматом построена на МК **STC8C8K64U**.

> ⚠️ Перфюмерный/спрей-автомат: спиральные моторы + распыление. **Управления холодильником в этом APK нет** (см. раздел 7).

---

## 1. Архитектура связи

Планшет общается с железом по **двум независимым UART, оба 115200 бод**:

| Порт | Скорость | Назначение | Где в коде | Драйвер |
|------|----------|------------|------------|---------|
| `/dev/ttyS1` | 115200 8N1 | **Управление автоматом** (моторы, монеты, версия, OTA платы) | C# `ThreadManager.cs`, `PortOpen("/dev/ttyS1",115200)` | `libserial_port.so` (P/Invoke) |
| `/dev/ttyS5` | 115200 | Оплата (протокол «Marshall»/bitmick VPOS, карты) | Java `com.lemon.marshall_sdk_library` | `libjSerialComm.so` |

**Для управления моторами/автоматом нужен `/dev/ttyS1`, 115200, 8N1.**
(Если при снятии логов идут «одни нули» — почти наверняка не та скорость/линия/порт.)

Нативные функции `libserial_port.so`:
`PortOpen, PortSet, PortClose, PortSend, PortRecv, ClearPortBuffer, ClearAllPortBuffer, GetPortInBufferLen`.

## 2. Общий формат кадра (ttyS1)

```
AA  [len]  [payload...]  [XOR]  DD
```

| Поле | Значение | Описание |
|------|----------|----------|
| Старт | `0xAA` (170) | начало кадра |
| len | 1 байт | длина/поле кадра |
| payload | N байт | команда + данные |
| XOR | 1 байт | контрольная сумма (XOR внутренних байт) |
| Стоп | `0xDD` (221) | конец кадра |

**Контрольная сумма** = XOR байтов начиная с индекса `1` до байта перед XOR (включительно).

```c
// как считает прошивка/приложение
uint8_t chk = 0;
for (int i = 1; i <= 14; i++)      // для 17-байтного кадра выдачи
    chk ^= frame[i];
frame[15] = chk;
```

## 3. Команда ВЫДАЧИ (крутить спираль) — основная

Метод `ThreadManager.A_Buy_Tongbushuju(row, column, perfume_time, vending_time)`.
Отправляется `PortSend(fd, data, 17)`. **17 байт:**

```
AA 0E 01 30 [row] [col] [t_hi] [t_lo] 00 00 00 00 [q] 00 00 [XOR] DD
```

| Индекс | Значение | Смысл |
|-------:|----------|-------|
| 0 | `AA` | старт |
| 1 | `0E` (14) | длина |
| 2 | `01` | команда = **выдача** (ответ платы = `02`) |
| 3 | `30` (48) | константа |
| 4 | `row` | **ряд/этаж** (1..N) |
| 5 | `col` | **колонка/слот** (1..N) |
| 6 | `t_hi` | время работы мотора, старший байт |
| 7 | `t_lo` | время работы мотора, младший байт (16 бит, **big-endian**) |
| 8..11 | `00` | нули |
| 12 | `q` = `01` | константа (флаг количества); **НЕ время** |
| 13,14 | `00` | нули |
| 15 | `XOR` | XOR байт [1..14] |
| 16 | `DD` | стоп |

> Адресация слота — **пара (ряд, колонка)**, не сквозной номер.
> Время выдачи задаётся в байтах **[6][7]** (big-endian). Дефолт в приложении — 200.
> На отладочном экране приложения байт [12] ошибочно подписан как «время» — это устаревшая подпись, реально там константа `1`.

Вызов в коде: `GameManager.cs` → `A_Buy_Tongbushuju(select_row, select_column, 1, vending_time)`.

### Готовые примеры

**Ряд 1, колонка 3, время 200 (0x00C8):**
```
AA 0E 01 30 01 03 00 C8 00 00 00 00 01 00 00 F4 DD
```

**Ряд 1, колонка 12, время 5:**
```
AA 0E 01 30 01 0C 00 05 00 00 00 00 01 00 00 36 DD
```

Проверка XOR для первого примера (байты [1..14]):
`0E^01^30^01^03^00^C8^00^00^00^00^01^00^00 = 0xF4` ✔

## 4. Ответ платы — результат выдачи

Плата присылает 17-байтный кадр с тем же форматом, но `[2]=0x02`:

```
AA 0E 02 30 ... [status] ... [XOR] DD
```

Проверка приёма: `[0]=AA, [1]=0E, [2]=02, [3]=30, [16]=DD`, `XOR[1..14]==[15]`.

**Байт [12] = код результата:**

| [12] | Значение |
|------|----------|
| `0` | **успех** (出货成功 — товар выдан, списать остаток) |
| `1` | ошибка (мотор не сработал / долго не распыляет) |
| `2` | ошибка (товар не зафиксирован) |

Разбор: `ThreadManager.Serial_Port_Get` → `Handle_Delivery_Result` (опрашивается каждый кадр в `Update`).

## 5. Счётчик монет (плата → планшет)

7-байтный кадр:

```
AA 03 30 [cnt_hi] [cnt_lo] [XOR] DD
```

- `[1]=03`, `[2]=30`
- накопительный счётчик монет = `cnt_hi*256 + cnt_lo`
- `XOR` считается по байтам [1..4]
- приложение берёт дельту относительно прошлого значения и шлёт в MQTT (`Publish_Coin_Count`)

---

## 6. Запрос версии платы (планшет → плата)

5-байтный кадр:

```
AA 02 80 82 DD
```

- `[1]=02` (длина), `[2]=80` (команда «версия»), `[3]=XOR(02^80)=82`, `[4]=DD`
- метод `ThreadManager.Hardware_version_btn`

## 7. Холодильник / охлаждение / температура

**Отсутствует.** Это перфюмерный/спрей-автомат (строки `香水机` «перфюм. машина», `Popcorn_`, `长时间不喷` «долго не распыляет»).
Поиск по всему C# и Java (`温度/制冷/压缩机/冷/加热/门/锁/灯/风扇`, `temperature/compressor/fridge/cool/door/heat/fan`) — **ноль совпадений** в контексте управления.

Если ваш реальный автомат **охлаждаемый** — нужна одна из опций:
1. APK именно от охлаждаемого автомата (там будет ещё команда с другим кодом `[2]`, тем же кадром `AA … XOR DD`) — найдём тем же методом.
2. Зондирование платы перебором кодов команд по этому же протоколу.

## 8. Обновление прошивки платы (OTA)

- Команда `0xCD` (205), пакетная передача `.bin` на плату по тому же `/dev/ttyS1`.
- Класс `ThreadManager.lyy_ota` (`file_packet_send`, `yxj_data_parse`).
- Файл прошивки `uy.bin` качается с:
  `https://appxianggan.oss-cn-hongkong.aliyuncs.com/uy.bin`
- То есть прошивку STC8 можно обновлять по UART, и сам файл прошивки доступен (полезно для более глубокого реверса, хотя это машинный код STC8).

## 9. Итоговая карта кадров ttyS1

Все кадры: 115200 8N1, обрамление `AA … DD`, контроль — XOR внутренних байт.

| Команда | Направление | Байт | Признак | XOR |
|---------|-------------|------|---------|-----|
| Выдача (крутить спираль) | планшет→плата | 17 | `[2]=01`, `[3]=30` | [1..14] → [15] |
| Результат выдачи | плата→планшет | 17 | `[2]=02`, `[12]`=статус | [1..14] → [15] |
| Счётчик монет | плата→планшет | 7 | `[1]=03`, `[2]=30` | [1..4] → [5] |
| Версия платы | планшет→плата | 5 | `[2]=80` | [1..2] → [3] |
| OTA прошивки | планшет→плата | пакеты | cmd `0xCD` | по кадру OTA |

## 10. Практика

1. Подключиться к `/dev/ttyS1`, **115200 8N1**.
2. Отправить кадр выдачи (раздел 3) — спираль (ряд, колонка) должна крутануться.
3. Дождаться ответа платы; `[12]=0` = успех.
4. В самом приложении есть скрытая отладка (`ThreadManager`: поля «ряд/колонка» + кнопки «start test» / «all test»), которая шлёт ровно эту команду — удобно сверять на живом железе.
