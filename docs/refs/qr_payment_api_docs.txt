7.   Authentication method 
In order to prevent the interface from being maliciously invoked, the server needs to verify 
the data transmitted to the interface. The specific methods are as follows.： 
1、Each machine is assigned an ID (called machid) and a key (appkey). The key is a string 
consisting of 16 upper and lower case letters or numbers. Different machines have different 
machids and keys. Define can be written in code or configuration file before leaving the factory. 
2、There are three additional parameters to be transferred when calling any interface： 
randstr：Random strings, also composed of 16 upper and lower case letters or numbers, should 
be regenerated every time the interface is called. 
timestamp：Machine local Unix timestamp, with a global standard time error of no more than  2 
minutes. 
sign：signature，generated as follows:
